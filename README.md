# Cloud-Native Transaction Monitoring & Fraud Detection Pipeline

That's a very long title but it's a Work-in-Progress

This project aims to build a production-style data platform that to teach me the basics.

This project will also try to
- Ingest a large volume of transaction data (generated manually or API if I can find one)
- Process and validate this data using Python and SQL
- Store said data in BigQuery
- Detect fraud patterns (Neo4j was recommended but subject to change if I find something better)
- Aims to be run on Google Cloud Platform (GCP) services
- Hopefully has CI/CD and automation
- Include data quality checks
